# Roadmap

Visão macro das entregas por fase. Detalhes de cada funcionalidade estão nos [Requisitos](../requirements/functional.md).

---

## 🗓️ Fases

### Fase 0 — Fundação ✅
**Objetivo:** Documentar e planejar antes de escrever uma linha de código.

- [x] Visão do produto
- [x] Personas
- [x] Estrutura de repositórios
- [x] Levantamento de requisitos
- [ ] Arquitetura técnica
- [ ] Wireframes

---

### Fase 1 — MVP 🔵 *(em planejamento)*
**Objetivo:** Lançar a versão mínima com o fluxo central de conversação funcionando.

**Funcionalidades:**
- [ ] Autenticação (e-mail + Google)
- [ ] Chat por texto com IA
- [ ] Correção de erros em tempo real
- [ ] 3 cenários de roleplay (texto)
- [ ] Histórico de conversas
- [ ] Plano Free com limites

---

### Fase 2 — Voz 🔒 *(bloqueado pela Fase 1)*
**Objetivo:** Adicionar a camada de voz ao produto.

- [ ] Input de voz (speech-to-text)
- [ ] Output de voz (text-to-speech)
- [ ] Roleplay por voz
- [ ] Correção de pronúncia

---

### Fase 3 — Engajamento 🔒
**Objetivo:** Aumentar retenção e conversão para Pro.

- [ ] Dashboard de progresso
- [ ] Sistema de streak
- [ ] Conquistas (badges)
- [ ] Plano Pro com pagamento

---

### Fase 4 — Expansão 🔒
**Objetivo:** Crescer base e explorar novos canais.

- [ ] App mobile (PWA ou nativo)
- [ ] Novos cenários de roleplay
- [ ] Modo de prática com tema livre
- [ ] Relatório semanal por e-mail

---

!!! warning "Este roadmap é vivo"
    As prioridades podem mudar com base em feedback de usuários e métricas de produto. Atualizações são registradas via Pull Request com justificativa.
